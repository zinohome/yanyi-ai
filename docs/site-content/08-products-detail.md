## IndustriaX 智能生产力底座

- 路径：`/products/industriax`
> 让企业专家知识被 AI 理解、调用、复用

面向先进制造企业自研的智能生产力底座，把散落的工程知识与业务流程，转化为可靠、可追溯、可审计的智能体工作流。

**概述**

IndustriaX 承载企业级知识库、智能体应用、业务流程与智能工作流决策能力，支持在客户内网环境私有化部署、可离线运行。小模型驱动、松散可组合，靠结构而非模型规模拿到工业级可靠性。

**挑战**

工程知识分散在邮件、文档、测试报告、设计手册与专家个人经验中，难以沉淀为组织能力；研发、质量、工艺、采购与生产之间存在信息断点，问题定位、变更影响判断与计划调整高度依赖人工经验。企业对数据安全、知识产权保护与私有化部署要求高，无法简单采用通用云端 AI。

**痛点**

- 隐性专家经验难以沉淀、传承与复用
- 跨部门信息断点，问题定位与变更评估靠人工
- 数据不能出厂，通用云端 AI 无法落地

**方案闭环**

- **专家知识资产化** — 解析工程文档、测试报告、8D/FA、变更记录、设计规范与工艺文件
- **业务与工程语义建模** — 围绕产品、零部件、失效模式、变更影响等建立可复用知识网络
- **智能工作流决策编排** — 将专家经验、业务规则、审批流程与 AI 能力组合为可执行、可追踪工作流
- **可靠可审计输出** — 所有关键输出保留来源引用、过程记录与版本痕迹，便于复核追溯

**核心功能**

- **专家知识资产化** — 对非结构化与半结构化工业资料进行解析、抽取、组织与检索。
- **智能工作流决策编排** — 服务 FA 分析、ECN/PCN 影响评估、测试验证、主计划与报告生成等场景。
- **私有化与可离线运行** — 支持客户内网部署，敏感数据不出厂，适配企业已有 IT 环境。

**价值 / 成效**

- **私有化部署** — 数据不出厂，核心知识资产留在内网
- **可追溯** — 来源引用 · 过程记录 · 版本痕迹
- **可复用** — 每个场景交付沉淀为可复用组件与方法

**适用对象**：研发 / 工艺工程师、质量与失效分析团队、生产计划与供应链、企业 IT 与数字化负责人

## IndustrialMind 智能体大脑

- 路径：`/products/industrialmind`
> 智能体的思考、组织与编排层

多智能体架构中的"大脑"——负责思考、人格与组织、记忆调度与工具路由，是会话与决策的唯一事实源。

**概述**

IndustrialMind 将语言理解、任务规划、记忆调度与工具调用决策统一编排：由它决定"想什么、记什么、调用哪个能力"，再把工具执行委托给底座的执行层，形成清晰的思考—执行分离。

**挑战**

把大模型直接接入业务系统，往往得到一个"能聊天但不可控"的黑盒：人格、记忆、历史与工具调度混在一起，行为难以约束、结果难以复核。工业与企业级场景需要一个职责清晰、状态可控的编排内核。

**痛点**

- 人格、记忆与工具调度耦合，行为难约束
- 会话事实源不唯一，结果难以复核
- 缺少清晰的多智能体分工

**方案闭环**

- **理解与规划** — 解析意图，拆解任务，规划执行路径
- **记忆调度** — 决定记什么、取什么，维护会话事实源
- **工具路由** — 把工具执行委托给专用执行层
- **人格与组织** — 统一维护人格提示与多智能体协作规则

**核心功能**

- **思考与语言编排** — 意图理解、任务规划与多轮对话统一编排。
- **记忆与路由** — 作为唯一会话事实源，调度记忆并决策工具调用。
- **思考与执行分离** — 思考归大脑、执行归底座执行层，职责清晰、可维护。

**价值 / 成效**

- **单一事实源** — 会话历史、记忆、人格集中管理
- **可组合** — 独立服务、按需替换
- **可审计** — 决策过程可记录、可回溯

**适用对象**：智能体 / 应用研发团队、企业 AI 平台团队

## IndustrialMemory 统一记忆平台

- 路径：`/products/industrialmemory`
> 一套 API 整合三大记忆引擎

统一 AI 记忆服务平台——单一 API 整合对话记忆、用户画像与知识图谱三大引擎，开箱即用的管理 UI 与多租户治理。

**概述**

IndustrialMemory 把对话记忆（从对话自动提取事实）、用户画像（结构化偏好与背景）、知识图谱（文档→实体关系→图检索）三类记忆整合为一套 REST + gRPC API，并内置鉴权、审计、备份与观测。

**挑战**

智能体要"越用越懂"，就需要长期、可治理的记忆；但对话记忆、用户画像与知识图谱各有专长、各自为政，逐个接入成本高，且难以统一鉴权与审计。

**痛点**

- 三类记忆各自为政，逐个接入成本高
- 缺少统一鉴权、审计与备份
- 权威知识与低可信记忆混淆

**方案闭环**

- **接入** — 对话、文档、画像数据统一入库
- **组织** — 事实提取、画像结构化、实体关系成图
- **检索** — 语义搜索与图检索，生成上下文
- **治理** — 多租户鉴权、审计、备份与观测

**核心功能**

- **对话记忆** — 从对话自动提取事实，支持语义搜索。
- **用户画像** — 结构化存储偏好与背景，生成上下文提示。
- **知识图谱** — 文档转化为实体/关系图，支持图检索。

**价值 / 成效**

- **单一 API** — REST + gRPC 整合三大记忆引擎
- **可治理** — 多租户鉴权 · 审计 · 备份
- **知识记忆分离** — 权威知识与低可信记忆分层

**适用对象**：智能体 / 应用研发团队、需要长期记忆的企业系统

## IndustrialVoice 实时语音底座

- 路径：`/products/industrialvoice`
> 全栈本地推理的自托管语音平台

自托管语音服务平台——实时语音转写、流式合成与音色克隆、端到端语音对话，全栈本地推理，单 GPU 即可运行。

**概述**

IndustrialVoice 提供 STT、TTS 与实时语音对话三类服务，通过标准 HTTP / WebSocket API 接入任意应用；内置鉴权、审计、用量监控与管理后台，docker-compose 一键启停，敏感语音数据无需出网。

**挑战**

语音是工业现场与一线最自然的交互方式，但云端语音服务面临数据合规与网络依赖问题；自建又受限于算力与工程复杂度，难以在私有环境稳定落地。

**痛点**

- 云端语音存在数据合规与网络依赖
- 自建语音栈算力与工程门槛高
- 缺少鉴权、审计与用量治理

**方案闭环**

- **实时转写 (STT)** — 流式语音转文字，低延迟接入
- **流式合成 (TTS)** — 流式语音合成，支持音色克隆
- **实时对话** — 端到端语音对话，多轮记忆与打断
- **治理** — 鉴权、审计、用量监控与管理后台

**核心功能**

- **全栈本地推理** — 单 GPU ≤ 12GB 即可运行，数据无需出网。
- **标准 API 接入** — HTTP / WebSocket 接入任意应用，附 Python SDK。
- **一键部署与治理** — docker-compose 启停，内置鉴权、审计与监控。

**价值 / 成效**

- **本地推理** — 单 GPU ≤ 12GB，数据不出网
- **三大服务** — STT · TTS · 实时语音对话
- **一键部署** — docker-compose 启停

**适用对象**：语音交互应用团队、对数据合规要求高的企业

## IndustrialVision 工业视觉智能底座

- 路径：`/products/industrialvision`
> 从看见异常，到理解异常为何发生

制造现场的智能感知与理解平台——以视觉语言（VL）模型与视觉识别技术，覆盖产品缺陷检测、装配状态识别、零部件识别、OCR、视觉质量分析与视频行为理解，并融合多模态数据。

**概述**

IndustrialVision 作为"感官"家族的视觉入口，与 IndustrialVoice 并列，为智能体提供看得懂现场的能力：识别产品缺陷、装配状态与零部件，读取图纸、单据与仪表中的文字与版面（OCR / 版面解析），理解质量异常与作业场景的语义（VL 模型与视频行为理解）。结果结构化后写入 IndustrialMemory，与 IndustrialMind 协同，把"看见异常"推进到"理解异常为何发生"——缺陷不只是被检出，更被归因、被反哺到设计与工艺。全栈可私有化部署，图像数据不出厂。

**挑战**

制造现场大量关键信息以图像形式存在——产品缺陷、装配状态、工程图纸、纸质单据、仪表盘、质检画面与作业视频。传统机器视觉只能按预设规则判定"有无缺陷"，说不出成因；人工录入慢且易错；通用云端视觉又面临数据合规与网络依赖，难以稳定接入业务系统。

**痛点**

- 机器视觉只判定有无缺陷，说不出为什么
- 图纸、单据、仪表信息靠人工录入，慢且易错
- 云端视觉存在数据合规与网络依赖
- 视觉结果难以结构化、难以反哺设计与工艺

**方案闭环**

- **感知** — 相机、扫描与作业视频多源接入
- **识别** — 缺陷、装配状态、零部件与 OCR 版面解析
- **理解** — VL 模型解释语义，归因异常成因
- **入库与反哺** — 结构化写入记忆平台，反哺设计与工艺；低置信转人工复核

**核心功能**

- **缺陷与装配识别** — 识别产品缺陷、装配状态与零部件，替代人工目检。
- **文档 OCR 与版面解析** — 高精度识别图纸、单据、报告中的文字、表格与版面结构。
- **VL 理解与视频行为** — 用视觉语言模型理解仪表读数、质量异常、作业场景与视频行为语义。
- **多模态融合与入库** — 融合视觉、时序与文档数据，规范化写入 IndustrialMemory。

**价值 / 成效**

- **看得懂现场** — 从检出缺陷到归因成因
- **私有化** — 图像数据不出厂
- **可追溯** — 保留来源与置信度，可复核

**适用对象**：质量与质检团队、研发 / 工艺工程师、制造现场与设备团队

**挑战**

**痛点**

- 科研任务拆解与推进全靠个人经验
- 文献与证据分析耗时，难以系统覆盖
- 工具割裂，检索、统计与写作各自为战

**方案闭环**

- **任务拆解** — 把科研目标拆为可执行、可追踪的任务链
- **多 Agent 协同** — 检索、分析、整理、生成分工协作
- **工具调用** — 统计、检索、文献库与项目管理工具编排
- **流程自动化** — 过程留痕，结果可复盘、可复用

**核心功能**

- **科研任务拆解与规划** — 把科研目标转化为可执行任务链，明确依赖与产出。
- **多 Agent 协同** — 临床科研、转化、实验室管理与医企合作 Agent 分工协作。
- **文献与证据分析** — 系统检索、筛选与综合，输出带出处的证据结论。

**价值 / 成效**

- **可编排** — 科研流程从人力推动转为可编排
- **有出处** — 证据结论保留引用，可复核
- **私有化** — 院内部署，数据不出院

**适用对象**：临床医生与研究团队、重点实验室 PI 与科研骨干、医院科研管理

## MedicalMemory 医学统一记忆平台

- 路径：`/products/medicalmemory`
> 让医疗机构拥有自己的医学知识记忆

医疗机构科研知识与经验资产底座——沉淀医学文献、临床指南、病种知识、专家经验、科研课题、临床案例、实验数据与转化项目资料。

**概述**

MedicalMemory 把机构内外的医学知识统一沉淀为可检索、可推理、可复用的记忆底座：外部的文献与指南，内部的病种知识、临床案例、实验数据、课题与转化项目资料，以及最难留存的专家经验。它是 MedicalMind 推理时的事实来源，也是机构科研能力不随人员流动而流失的保障。

**挑战**

医学知识的半衰期短、来源分散：指南在更新，文献在爆炸，而机构自己的病种经验、临床案例与课题积累往往只存在于个别专家的记忆和硬盘里。人员一流动，机构的科研能力就跟着流失。

**痛点**

- 专家经验只在个人手里，人走知识走
- 文献、指南、病历与课题资料分散在各系统
- 知识无法被 AI 直接理解与推理

**方案闭环**

- **汇聚** — 文献、指南、病种知识、案例、实验数据与课题资料统一接入
- **结构化** — 抽取实体与关系，构建医学知识网络
- **沉淀经验** — 把专家判断与决策依据数字化留存
- **供给推理** — 作为 MedicalMind 的事实来源，带出处返回

**核心功能**

- **多源知识汇聚** — 文献、指南、病种知识、临床案例、实验数据与转化项目资料统一沉淀。
- **专家经验数字化** — 把专家的判断依据与决策过程留存为可复用知识。
- **可推理的知识网络** — 围绕病种、证据、课题与成果构建可检索、可推理的关系网络。

**价值 / 成效**

- **机构记忆** — 知识不随人员流动而流失

## MedicalVision 医学视觉智能底座

- 路径：`/products/medicalvision`
> 让影像与实验图像成为可推理的证据

面向医学影像、实验数据与临床视觉信息的智能感知底座——医学影像理解、图像辅助分析、实验图像识别、病理图像分析与多模态医学数据融合。

**概述**

MedicalVision 作为"感官"家族的视觉入口，与 MedicalVoice 并列：理解医学影像与病理图像的语义，识别实验图像中的目标与量化指标，并把视觉结论与文献证据、临床数据在 MedicalMemory 中融合，供 MedicalMind 推理时调用。它服务于科研而非诊断——输出的是可追溯、可复核的分析证据，最终判断始终由医生与研究者作出。全栈可私有化部署，影像数据不出院。

**挑战**

医学影像、病理切片与实验图像里承载着最密集的证据，却最难被批量利用：人工阅片与计数耗时，量化标准因人而异，图像结论又难以和文献证据、临床数据关联起来形成可复用的科研资产。云端影像分析则直接触碰患者数据合规。

**痛点**

- 人工阅片与计数耗时，量化标准因人而异
- 图像结论难以与文献证据、临床数据关联
- 云端影像分析触碰患者数据合规

**方案闭环**

- **接入** — 医学影像、病理切片与实验图像多源接入
- **识别与量化** — 目标识别、区域分割与指标量化
- **语义理解** — 理解影像与病理图像语义，给出带依据的分析
- **融合与复核** — 与文献、临床数据在 MedicalMemory 中融合；结论交专家复核

**核心功能**

- **医学影像理解** — 理解影像语义，辅助定位、分割与量化分析。
- **病理与实验图像分析** — 识别病理切片与实验图像中的目标，输出可复现的量化指标。
- **多模态医学数据融合** — 把视觉结论与文献证据、临床数据在记忆平台中关联。

**价值 / 成效**

- **服务科研** — 输出分析证据，最终判断由医生作出
- **可复现** — 量化标准统一，结果可复核
- **私有化** — 影像数据不出院

**适用对象**：临床研究与影像团队、病理与检验科室、重点实验室与科研团队
